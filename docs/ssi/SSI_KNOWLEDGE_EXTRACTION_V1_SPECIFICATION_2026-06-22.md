# 1. Purpose

Knowledge Extraction V1 derives reusable scientific knowledge from the deterministic outputs generated by previous SSI layers.

Unlike previous layers, Knowledge Extraction no longer describes individual runtime objects. Instead, it aggregates observations across all trajectories and forecasts to produce validated scientific knowledge.

Knowledge Extraction performs no decision making.

---

# 2. Scientific Motivation

Previous SSI layers answer the questions:

* What happened?
* How did the system evolve?
* What is the predicted next state?

Knowledge Extraction introduces the next scientific abstraction level:

What general scientific knowledge can be derived from the complete runtime?

The objective is to transform deterministic observations into reusable scientific knowledge.

---

* External scientific models

Knowledge Extraction V1 intentionally relies only on deterministic SSI outputs.

# 5. Scientific Objects

Internal object

KnowledgeCandidate

Knowledge Candidates represent possible scientific findings.

Knowledge objects represent validated scientific knowledge.

---

# 6. Knowledge Candidate

Knowledge Candidates are generated directly from deterministic statistics.

They are not yet considered validated scientific knowledge.

Typical examples include

* repeated state behaviour
* trajectory behaviour
* transition behaviour
* forecast behaviour

---

# 7. Knowledge Validation

Knowledge Validation evaluates every candidate before it becomes scientific knowledge.

Knowledge Extraction V1 validates

* deterministic reproducibility
* internal consistency
* support values
* finite numeric values
* valid references

Future versions may additionally validate

* statistical significance
* cross-runtime stability
* cross-dataset reproducibility
* Bayesian confidence
* uncertainty propagation

---

# 8. Knowledge Object

Each validated Knowledge object contains

Identity

* knowledge_id
* knowledge_type

Scientific description

* description

Support

* support_count
* support_ratio

Scientific quality

* confidence
* scientific_score

Metadata

* runtime_id
* knowledge_version

---

# 9. Scientific Scope

Knowledge Extraction V1 extracts only deterministic descriptive knowledge.

Examples include

* repeated state frequency
* forecast method distribution
* trajectory length characteristics
* transition characteristics

Knowledge Extraction V1 does not generate scientific rules.

---

# 10. Layer Separation

Knowledge Extraction shall not

* modify trajectories
* modify transitions
* modify forecasts
* optimize parameters
* classify regimes
* execute trades
* generate trading decisions

Its only responsibility is scientific knowledge extraction.

---

# 11. Determinism

Identical scientific inputs shall always produce identical Knowledge objects.

No stochastic behaviour is permitted.

No random sampling is permitted.

---

# 12. Engineering Principles

Knowledge Extraction shall reuse the common SSI infrastructure.

Required base classes

* ScientificObject
* ScientificResult
* ScientificProcessor
* ScientificRenderer
* ScientificPersistence
* ScientificArtifacts

No duplicated infrastructure shall be introduced.

---

# 13. Validation

Knowledge Extraction shall validate

* every Knowledge object originates from at least one Knowledge Candidate
* every Knowledge object is deterministic
* support values are valid
* confidence values are finite
* scientific scores are finite
* runtime identifiers are consistent

---

# 14. Acceptance Criteria

Knowledge Extraction V1 is accepted when

* Knowledge Candidates are generated
* Knowledge Validation succeeds
* validated Knowledge objects are produced
* renderer generates all artifacts
* persistence stores all artifacts
* end-to-end execution succeeds
* engineering gates pass
