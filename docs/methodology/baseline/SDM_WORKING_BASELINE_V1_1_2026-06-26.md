# Validated Methodological Rules

## Rule 1

Characteristics are not Necessary Properties.

Characteristics describe observed scientific properties.

Necessary Properties describe only the minimal scientifically required properties.

The two concepts shall always remain separated.

---

## Rule 2

Every Necessary Property shall undergo an explicit Necessary Property Validation before becoming part of the Property Configuration.

---

## Rule 3

Every Scientific Capability shall produce exactly one validated minimal Property Configuration.

Alternative Property Configurations require independent scientific justification.

---

## Rule 4

Scientific State Recognition shall always be derived directly from the validated Property Configuration.

Scientific States shall never introduce additional scientific assumptions.

---

## Rule 5

Capability boundaries shall be defined explicitly.

Each Capability shall document both:

Included Scientific Responsibility

and

Explicitly Excluded Scientific Responsibility.

Capability boundaries are considered a mandatory part of scientific specification.

---

# Governance

Scientific capabilities shall evolve only through explicit scientific justification.

Architectural convenience shall never justify new scientific capabilities.

New methodological rules shall not be introduced after a single capability pilot.

Working Baseline updates require repeated evidence across multiple completed capability pilots.
